# Requirements Document

## Project Description (Input)
ウェルカムページではなくトップページは真っ白な画面にしたい

## Introduction
本機能は、アプリケーションのトップページ（ルートパス `/`）を、既存のウェルカムページコンテンツから真っ白な画面に変更することを目的とします。これにより、ユーザーがトップページにアクセスした際に、装飾やコンテンツのない最小限の空白画面が表示されます。

## Requirements

### Requirement 1: トップページの空白表示
**Objective:** ユーザーとして、トップページ（`/`）にアクセスした際に、真っ白な画面が表示されることを期待します。これにより、シンプルで最小限の初期画面を提供できます。

#### Acceptance Criteria
1. When ユーザーがルートパス（`/`）にアクセスしたとき、the Top Page は背景色が白（`#FFFFFF`）の空白画面を表示する
2. The Top Page は既存のウェルカムページのコンテンツ（ヘッダー、ナビゲーション、メインコンテンツ、フッターなど）を表示しない
3. The Top Page は最小限の HTML 構造のみを含む
4. When ダークモードが有効な場合でも、the Top Page は白背景を維持する（または適切なダークモード対応を実装する）

### Requirement 2: 既存ウェルカムページの処理
**Objective:** 開発者として、既存のウェルカムページコンポーネントを適切に処理したいです。これにより、コードベースの整理と将来の再利用性を確保できます。

#### Acceptance Criteria
1. When トップページが空白画面に変更されたとき、the System は既存のウェルカムページコンポーネント（`resources/js/pages/welcome.tsx`）を削除または置き換える
2. If 既存のウェルカムページが他のルートで参照されている場合、the System は参照エラーを発生させない
3. The System はルート定義（`routes/web.php`）でトップページ（`/`）が新しい空白ページコンポーネントをレンダリングすることを保証する

### Requirement 3: トップページから遷移できなくなったページの削除
**Objective:** 開発者として、トップページから遷移できなくなったページを削除したいです。これにより、不要なコードを削除し、コードベースを整理できます。

#### Acceptance Criteria
1. When トップページが空白画面に変更されたとき、the System はウェルカムページからリンクされていたページ（例：Dashboard ページ）を削除する
2. The System は削除対象のページコンポーネント（`resources/js/pages/dashboard.tsx`）を削除する
3. The System は削除対象のページに関連するルート定義（`routes/web.php` の `dashboard` ルート）を削除する
4. If 削除対象のページが他のコンポーネントやルートから参照されている場合、the System は参照エラーを発生させない（参照を削除または更新する）
5. The System は削除対象のページに関連する型定義やインポート文を削除する
